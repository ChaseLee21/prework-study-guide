HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The meta tags add important information for the web browser that is invisible to the user
  • The header tag is inside the body tag and will always display content at the top of the page
  • The footer tag is inside the body tag and will always display content at the bottom of the page
  • The alt attribute in an img tag helps screen readers know what the image is displaying
  • h1 h2 h3 h4 h5 h6 tags change the size of text that you would like to be "headings"
  • heading tags are important to use in order so the screen reader knows which one to read first

CSS

  • css stands for cascading style sheet and helps add style to the webpage
  • A margin indicates how much space we want around the outside of an element.
  • A border lies inbetween the margin and padding and can be any width and color you would like
  • A padding indicates how much space we want around the content inside an element.
  • The text-align attribute specifies where you want the text aligned within a given element
  • The display attribute for elements specifies how the element should be displayed on the web page and have two main values, block / inline

Git

  • Git helps us keep track of our code versions
  • git status: checks what branch we are currently working in
  • git checkout -b branch-name: creates a new branch and switches to the branch-name
  • git checkout branch-name: switches to the branch-name
  • git add -A: adds all files to the staging area, a preparation area before you commit the changes
  • git commit -m "commit message goes here": commits the staged files to the repository creating a "snapshot" of the code
  • git pull origin branch-name: pulls the current code from the remote repository in the specified branch-name
  • git push origin branch-name: pushes the current code to the remote repository in the specified branch-name

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • JS adds functionality to the webpage and helps the user interact with the page
  • JS adds conditional statements to the code to make decisions based on data and/or user input
  • A code block is a section of code that is sectioned off between to curly braces { code goes here }
  • An if statement takes a conditional statement and based on the results being truthy or falsey executes a code block
  • A for loop repeats a code block until the conditional statements is falsey
